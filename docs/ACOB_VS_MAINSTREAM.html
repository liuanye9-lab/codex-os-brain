Agentic Coding OS Brain

ACOB is not another vector memory store. It is a Codex-facing agentic coding harness.

Mainstream systems are strong at memory extraction, graph memory, context repositories, or state-machine orchestration. ACOB connects memory to coding execution: working memory, dispatch, verification, reward, replay, lifecycle, and dashboard evidence.

Mem0 Memory extraction / retrieval

Strong fit for selective long-term memory, personalization, and reducing repeated context injection.

Zep / Graphiti Temporal graph memory

Strong fit for entity relationships, time-aware facts, provenance, and graph traversal.

Letta Context repo / memory OS

Strong fit for agent-managed context files, durable memory operations, and repository-like memory workflows.

LangGraph State machine orchestration

Strong fit for graph workflows, state transitions, human-in-the-loop, and production agent control flow.

ACOB Codex-facing agentic coding harness

Strong fit for coding tasks where memory must feed task cards, context packs, tool calls, verification, reward, replay, and public-safe release gates.

1. Working Memory

Scope the current goal, constraints, focus files, and risks.

2. Retrieval Pipeline

Rewrite query, recall candidates, rerank by evidence and freshness.

3. Context Pack

Inject only bounded public-safe evidence into the agent turn.

4. Dispatch Gate

Use sub-agents only when ROI, verifiability, and privacy allow it.

5. Verification

Do not claim completion without checks, evals, or observable evidence.

6. Replay

Turn trace, eval, and reward into candidates, not automatic self-modification.

Honest boundary: ACOB does not claim to outperform mature memory databases today. Its advantage is the integrated coding harness around memory: context, agents, tools, verification, replay, and public-safe governance in one local install.
